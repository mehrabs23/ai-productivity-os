# Day 3: Operationalizing Your Life OS — Dashboards, Workflow, Reminders, and Daily Check-ins

## Building a Personal Productivity System with AI

---

### Day 3 Goal

By the end of Day 3, every learner has operationalized their Life OS and can use it to navigate a real day — through a functional dashboard, a visual workflow, reminder-ready views, a daily check-in routine, and a weekly status view.

### Day 3 Transformation

**Before Day 3:** The Life OS has good databases, but learners still navigate by memory, feel overwhelmed by open tasks, and lack a routine for using the system daily.

**After Day 3:** The Life OS is a living, usable system. Learners open a single dashboard each morning, see what matters, capture what comes in, check off habits, and close the day with a brief review. The system runs — not just exists.

### Quick Frame: What Is a Life OS Dashboard?
A Life OS dashboard is not a project management tool. It is a personal command centre that surfaces what matters across all life areas — work, study, personal goals, health habits, admin, and routines — so you can make better decisions about where your time and energy go.

### Life Areas to Consider (Encourage Broad Thinking)

## Session 2: Life OS Workflow and Kanban (45 minutes)

### Teaching Portion (15 minutes)

**Concept: A unified Life OS workflow**

One critical decision in an effective Life OS: use a single, consistent set of workflow statuses across **all** tasks — personal, professional, academic, and everything in between. This avoids a system where your work tasks have different stages than your personal tasks, which quickly creates a fragmented experience.

**Recommended Life OS workflow statuses:**

| Status | What Goes Here |
|--------|---------------|
| **Inbox** | Captured but not yet processed |
| **Planned** | Acknowledged and queued, but not scheduled |
| **This Week** | Committed for the current week |
| **In Progress** | Actively being worked on (limit 3 at a time) |
| **Waiting / Blocked** | Can't move — waiting on someone or something |
| **Done** | Completed |

**Why a Kanban board for personal life?**

Kanban is not just a corporate tool. When your board shows tasks from all life areas in a single view — your upcoming dentist admin next to your essay draft next to your content idea — you start to see where energy is actually flowing and where it is stuck.

Visual information is faster to process than a list. A blocked task looks different from an in-progress task. A full "This Week" column signals overcommitment before it becomes a crisis.

### Demo (15 minutes)

**Instructor builds Alex's Life OS Kanban:**

1. Open Alex's Task Database → Add a new view → Select "Board"
2. Group by: Status property
3. Edit Status options to match the six recommended statuses above (assign distinct colors)
4. Move Alex's tasks into correct columns: personal goal tasks into Inbox/Planned, work tasks into In Progress and This Week, blocked admin task into Waiting/Blocked
5. Add an Area/Life Domain property if not present (Select type): Work / Studies / Personal / Content / Health / Admin / Finance
6. Create filtered Kanban view "Active" — exclude Inbox and Done
7. Add a WIP limit Callout block: "🎯 In Progress limit: 3. If you're adding a 4th, something must be moved to Done or Planned first."

**Instructor moment:** "Look at this board. Alex has tasks from 4 life areas all visible at once. This is a life system — not just a work system."

### Student Task 2A: Build Your Life OS Kanban (30 minutes)

**Minimum Viable Task:**
1. Open your Task Database → create a Board view grouped by Status
2. Set up 6 status columns: Inbox / Planned / This Week / In Progress / Waiting / Done
3. Assign distinct colors so columns are immediately recognisable
4. Add Area/Life Domain property if not yet present
5. Move existing tasks into correct columns based on their real current state
6. Create "Active Work" filtered view: exclude Inbox and Done
7. Add WIP limit callout above the board

**Success looks like:** A board with tasks distributed across multiple life areas, not just work.

**Common mistakes:**
- Having 8+ tasks in "In Progress" (violates WIP limit)
- Only populating work tasks (system becomes narrow)
- Not creating the filtered "Active Work" view (board stays noisy)

**Stretch version:** Add a second board view grouped by Area/Life Domain to see which life areas have the most open work.

**Deliverable:** Task Database in Board view with 6 status columns, tasks from 2+ life areas, "Active" filtered view.

## Session 3: Build the Life OS Dashboard (60 minutes)

### Teaching Portion (15 minutes)

**Concept: The dashboard as daily operating layer**

The dashboard is not a decoration. It is the page you open every morning instead of your task list, your email, or your notes app. It answers three questions in under 10 seconds:
1. What matters most today?
2. What is coming soon and might be forgotten?
3. Am I making progress across the areas that matter to me?

**Dashboard design principles:**
- **Action over aesthetics.** The most important view is at the top. Decoration is in the middle or below the fold.
- **Linked views, not copies.** Everything on the dashboard is pulled from your existing databases — nothing is duplicated.
- **Narrow filters.** Show exactly what you need to act on — not everything that exists.
- **One click to action.** From the dashboard, you should be able to open any task or project in one click.

**Required dashboard sections:**

| Section | Purpose |
|---------|---------|
| Today | Tasks due or prioritised for today |
| This Week | Tasks due within 7 days |
| Active Projects / Areas | Active items grouped by area |
| Kanban / Workflow | Embedded board view of current status |
| Due Soon | Tasks due in the next 3–5 days |
| Overdue | Tasks where due date has already passed |
| Completed Recently | Tasks completed in the last 7 days (for momentum) |
| Capture / Inbox | Quick entry for new tasks or ideas |
| Daily Check-In | Daily reflection and intention section |
| Weekly Status | Progress view for weekly review |

### Demo (20 minutes)

**Instructor builds Alex's Life OS Dashboard:**

1. Create new Notion page: "🏠 Alex's Life OS"
2. Enable full-width layout (click page •••  → "Full width")
3. Add icon (🏠) and cover photo (from Notion's free library — original screenshot note: ⟨add your own cover later⟩)
4. Add header callout: "Life OS | [Life Area icons] | Week of [date]"
5. Create 2-column layout (drag blocks side by side)
6. **Column 1, top:** Linked database "🎯 Today" — Task DB, filter: Due = today OR Status = In Progress, gallery or list view, show: Name + Area + Status only
7. **Column 1, middle:** Linked database "📅 This Week" — Task DB, filter: Due ≤ today + 7, sort: Priority high → low
8. **Column 2, top:** Linked database "⚠️ Overdue" — Task DB, filter: Due < today AND Status ≠ Done, sort: Due date ascending
9. **Column 2, middle:** Linked database "🔔 Due Soon" — Task DB, filter: Due = in next 3 days AND Status ≠ Done
10. **Below fold (full-width):** Linked database "🗂️ Kanban — Active Work" — Board view, filtered to exclude Inbox/Done
11. **Below fold:** "✅ Completed Recently" — Task DB, filter: Status = Done AND Completion date ≥ last 7 days
12. **Sidebar area (or bottom column):** "🏷️ Active Areas" — Goals/Projects DB, filter: Status = Active

**Instructor pause:** "This took about 8 minutes when you know the steps. The next 30 minutes is yours to do the same."

### Student Task 3A: Build the Life OS Dashboard (40 minutes)

**Minimum Viable Task:**
1. Create a new Notion page with an icon, cover, and full-width layout
2. Name it: "[Your emoji] Your Name's Life OS" (make it yours)
3. Add a header callout with today's date and your active life areas
4. Set up a 2-column layout and add these linked database views (minimum 4):
   - Today / In Progress
   - This Week
   - Due Soon
   - Overdue
5. Add Kanban / Active Work view below the fold
6. Add Completed Recently below the fold for momentum visibility
7. Run the 5-Second Test: open dashboard from a fresh tab — can you see today's priority in under 5 seconds?

**Success looks like:** A single page that meaningfully represents your current life commitments — not just your work projects.

**Common mistakes:**
- Adding too many sections (more than 8–9 becomes scroll-hell)
- Using Table view for every section (boring and dense — vary view types)
- Forgetting to set the page as your Notion home
- Making the page look pretty before it shows real data

**Stretch version:** Add a "navigation bar" at the top using button blocks or Notion's sidebar links to jump to different areas.

**Deliverable:** Life OS Dashboard page with 4+ linked views, icon, cover, 2-column layout.

## Session 4: Reminders and Due-Date Visibility (30 minutes)

### Teaching Portion (10 minutes)

**Concept: Why reminders matter**

One of the most common reasons productivity systems fail is that they are great repositories of information but terrible at surfacing urgency. Reminders reduce mental load by ensuring that due dates and approaching deadlines appear *when they matter* — not just when you happen to browse your task list.

**Two key reminder mechanisms in Notion:**

1. **Due Date property:** A Date property on each task. When set, it becomes the basis for "Due Soon" and "Overdue" filter views.

2. **Reminder Date property:** A secondary Date property used to trigger notifications *before* the due date. For example: a task due Friday can have a Reminder Date of Wednesday.

**Best practice:** Set reminders for tasks that are easy to forget because they are important but not urgent. Don't reminder-stamp every task — save reminders for the things that would genuinely be missed.

**Notion's built-in reminder:** When you set a Date property, click the date → enable the reminder option (e.g., "30 minutes before", "1 day before"). Notion will send a notification.

### Demo (10 minutes)

**Instructor sets up Alex's reminder views:**

1. Open Alex's Task database → Open a task that has no Due Date → Add one
2. Click the Due Date → enable the Notion reminder → set to "1 day before"
3. Add a "Reminder Date" property to the Task database (Date type)
4. Show the "Due Soon" view: filter = Due Date is within the next 3 days AND Status ≠ Done
5. Show the "Overdue" view: filter = Due Date is before today AND Status ≠ Done, sort by Due Date ascending
6. Add both views to the dashboard for immediate visibility

### Student Task 4A: Set Up Reminder Views (20 minutes)

**Minimum Viable Task:**
1. Add a "Reminder Date" property to your Task database (Date type)
2. Ensure at least 5 tasks have Due Dates set (add them now if missing)
3. Set a Notion reminder on 2–3 tasks (click the date → toggle reminder)
4. Create "Due Soon" filtered view: Due Date next 3 days, Status ≠ Done
5. Create "Overdue" filtered view: Due Date past, Status ≠ Done, sorted by Date
6. Add both views to your Life OS dashboard

**Success looks like:** Opening your dashboard and seeing immediately which tasks are overdue and which are coming up within 3 days — without searching.

**Common mistakes:**
- Not adding Due Dates to tasks (no data for filters to work on)
- Creating a "Due Soon" view that shows too many days ahead (keep it to 3–5 days max)
- Over-reminding: every task with a reminder = reminder fatigue

**Stretch version:** Add a "Reminder Date" filter view as a third reminder layer — tasks where Reminder Date = today.

## Session 5: Daily Check-In Routine (30 minutes)

### Teaching Portion (10 minutes)

**Concept: The Daily Check-In creates daily intention**

The single highest-leverage habit in any productivity system is a brief daily check-in — not just reviewing tasks, but deliberately setting an intention for the day. A check-in takes 5–10 minutes and transforms the system from a passive database into an active daily ritual.

**Why it works:**
- Forces prioritization at the start of each day before reactivity sets in
- Creates a moment of genuine self-honesty ("Was I avoiding something yesterday?")
- Builds a log of how productivity feels over time

**Check-in options (choose one that works for your learners):**

**Option A: Daily Check-In Section on Dashboard**
Add a Toggle block to the top of the dashboard. Each day: open the toggle, write 3–5 quick answers, collapse.

**Option B: Daily Check-In Database**
A simple Notion database (Date, Answers, Energy, Notes) with one entry per day.

**Option C: Dedicated Notion Page**
A recurring daily template page with consistent prompts.

**Recommended Daily Check-In questions:**
1. What are my top 3 priorities for today?
2. Is there anything overdue I need to address first?
3. Was there anything I avoided yesterday that I should do today?
4. What is my energy level today? (1–5)
5. What would make today feel complete?

### Demo (10 minutes)

**Instructor builds Alex's Daily Check-In on the dashboard:**

1. At the top of the dashboard, above the "Today" section, add a Toggle block: "📝 Today's Check-In — [Date]"
2. Inside the toggle, add 5 questions as text prompts with space to write
3. Answer each question for today using Alex's fictional context
4. Show how the toggle collapses/expands — keeps the page clean
5. Optionally: show the lightweight database option for learners who prefer logging

**Say to learners:** "You'll fill this in during the closing session today for the very first time."

### Student Task 5A: Create Your Daily Check-In (20 minutes)

**Minimum Viable Task:**
1. Add a "📝 Daily Check-In" toggle at the top of your Life OS Dashboard
2. Include at minimum these 3 questions inside:
   - What are my top 3 priorities today?
   - What (if anything) is overdue that I need to handle first?
   - What would make today feel complete?
3. Fill it in right now for today (this is your first real check-in)
4. Optional: Add Energy (1–5) and "Yesterday I avoided…" prompts

**Success looks like:** A check-in that takes under 5 minutes and guides your next action immediately after closing it.

**Common mistakes:**
- Making the check-in so long it becomes a journaling exercise (keep it under 5 minutes)
- Creating the section but not completing the habit of opening it
- Keeping it below the fold (it should be the first thing you see)

**Stretch version:** Create a separate "Daily Log" database with one entry per day. Link it to the dashboard.

## Session 6: Weekly Status / Progress View (30 minutes)

### Teaching Portion (10 minutes)

**Concept: Visibility across a week**

While the daily check-in handles today, the weekly status view handles the broader question: "Am I making progress across my life OS?" This view also sets up Day 4's weekly review — which will use it as source data.

**What a weekly status view contains:**
- Tasks completed this week (filtered Completed view, date ≥ start of week)
- Tasks still open from last week (overdue, not completed)
- Progress across active projects/areas
- A brief written weekly status summary

**The progress summary is not just data — it's a communication tool.** When shared with a partner, accountability buddy, or team, the weekly status view becomes a shareable progress snapshot.

### Demo (10 minutes)

**Instructor builds Alex's Weekly Status section:**

1. Below the main dashboard section, add a heading: "📊 Week in Review"
2. Add linked database: "Completed This Week" — Task DB, filter: Status = Done AND Completion Date ≥ start of current week
3. Add linked database: "Still Open from Last Week" — Task DB, filter: Due Date < today AND Status ≠ Done (reuse Overdue view)
4. Add a text block with a simple template:
   - **Week of:** [date]
   - **Wins this week:** 
   - **Still in progress:**
   - **What I'm reprioritising:**
   - **Focus for next week:**
5. "You'll fill in the text portion during the weekly review on Day 4. Today, you're building the container."

### Student Task 6A: Build Weekly Status Section (20 minutes)

**Minimum Viable Task:**
1. Add a "📊 Weekly Status" section to your Life OS Dashboard (below the main views)
2. Create "Completed This Week" view (Status = Done, Completion Date this week)
3. Add a Weekly Status text block with 4–5 prompt lines
4. Fill in today's "Wins so far this week" section with real data

**Success looks like:** A section that would allow you — or anyone else — to get a clear picture of the week's progress in under 2 minutes.

**Common mistakes:**
- Making the status block too long (5 lines max, not a journal)
- Not filtering by "this week" (shows all completed tasks, not useful)
- Skipping the written section (the data alone doesn't tell the story)

**Stretch version:** Add a "Next Week Focus" section so the transition into weekly planning feels prepared.

## Session 7: Simulate a Real Day Inside the Life OS (20 minutes)

### Instructor Actions

This session is entirely experiential — no building, just using the system you've built.

**Instructor-led walkthrough script:**

"Imagine it's tomorrow morning, 8:30am. You open Notion. Let's walk through the first 10 minutes of your day using only your Life OS dashboard."

1. **Open the dashboard.** What is the first thing you see?
2. **Read the Daily Check-In.** Fill it in (or open yesterday's and copy it to update).
3. **Scan the Today view.** Pick your #1 priority task. Open it. Take one action.
4. **Check the Overdue view.** Is there anything that needs to be handled first?
5. **Glance at Due Soon.** Note anything coming within 3 days.
6. **Check the Kanban — Active view.** Is everything in the right column?
7. **Capture one new task** that has come to mind during the simulation. Does it land in Inbox?

**Debrief questions:**
- How long did that take? (Target: under 10 minutes)
- What felt clear? What felt confusing?
- What would you change about your dashboard layout?

## Live-Teaching Notes

- **Pacing risk:** Dashboard builds take longer than expected for beginners. Protect 55–60 minutes for Session 3.
- **Aesthetic trap:** When students start choosing colors for columns instead of populating tasks, redirect with: "Your Kanban should have real data before it has beautiful colors."
- **Technology frustration:** Some learners will have Notion loading slowly. Put a break between Sessions 3 and 4 to give Notion time to sync.
- **Life OS framing:** Repeat the "Life OS" language several times. This is the reframe that makes the dashboard feel meaningful and personal, not just professional.
- **Check-In habit:** Make the check-in feel like a ritual, not a task. Model it: go through it live with Alex's context.

## Common Beginner Mistakes

| Mistake | How to Catch It | How to Fix It |
|---------|----------------|---------------|
| No Due Dates on tasks | "Due Soon" and "Overdue" views show 0 rows | Add due dates to 5+ tasks before building views |
| Kanban has only work tasks | Board looks like a work PM tool | Add personal, admin, and learning tasks |
| Dashboard is 12+ sections | Overwhelm on scroll | Apply 5-second test; remove anything that fails |
| Daily check-in below the fold | Never seen → never done | Move check-in toggle to the top of the page |
| Status values differ from recommended | Board doesn't group correctly | Standardize to recommended 6-value workflow |
