# System Optimization Prompts

## Building a Personal Productivity System with AI — Prompt Library

---

### When to Use This Category

Use these prompts when your system needs tuning, not rebuilding. Good for Day 3, Day 4, and the weeks after the course when the rubber meets the road.

System optimization prompts help answer:
- "My system exists but I never use half of it — what should I cut?"
- "I'm using the system but not feeling the benefits — what's wrong?"
- "My dashboard is cluttered and I dread opening it — how do I fix it?"
- "I'm inconsistent with my rituals — how do I make them stick?"
- "I built this thing but I don't know if it's actually helping my goals"

### When NOT to Use These Prompts

Do not use optimization prompts on Day 1 or Day 2 — you don't have enough data yet. You want at least 3–5 days of system use before running a meaningful audit.

## Section 1: System Health and Audit

---

### Prompt 6.1 — Life OS System Retrospective Prompt

**Purpose:** Run a structured health check on your full Life OS — identify what's working, what's been abandoned, and what single change would have the highest impact.

**When to use:** Day 4, Session 3 — and as a monthly maintenance ritual thereafter.

---

```
You are a productivity system consultant running a full system audit. I've been using my Life OS for [X weeks/months] and I want an honest evaluation of what's working and what should change.

My system components and current usage (mark as: Using consistently / Sometimes / Rarely / Not using):
- Goals Database: [STATUS]
- Projects Database: [STATUS]
- Task Database: [STATUS]
- Milestone Map: [STATUS]
- Anti-To-Do Log: [STATUS]
- Work Session Log: [STATUS]
- Attention Budget: [STATUS]
- Kanban Board: [STATUS]
- Life OS Dashboard: [STATUS]
- Daily Check-In: [STATUS]
- Streak / Habit Tracker: [STATUS]
- Daily Planning Routine (ChatGPT): [STATUS]
- Weekly Review: [STATUS]

Rituals I maintain consistently: [LIST]
Rituals I've dropped or struggle with: [LIST]
The component most likely to degrade the whole system if ignored: [NAME IT]
Time since I last meaningfully improved the system: [DURATION]

Please:
1. Identify the 2–3 components that are underused — and for each, recommend: keep and simplify / actively restart / retire
2. Identify the single highest-leverage change that would make the system more reliable or useful
3. Flag any component that is likely creating friction rather than reducing it — and why
4. Identify any component I'm missing that, based on my usage pattern, would meaningfully improve my output
5. Ask me one question that would help me make the most important decision about this system's future
```

### Prompt 6.2 — Life OS Dashboard Audit Prompt

**Purpose:** Get structured feedback on your Notion dashboard's usability by describing its layout and asking for a diagnostic assessment.

**When to use:** Day 3, Session 4 — and any time your dashboard feels cluttered, confusing, or unused.

---

```
You are a UX-trained information architect reviewing my personal Life OS dashboard in Notion. I want structured feedback on whether it's actually useful or just impressive-looking.

My dashboard layout (describe sections from top to bottom):
[DESCRIBE EACH SECTION — name, what database it links to, what filter/sort is applied]

Example format:
1. "Daily Check-In" — toggle block, no database
2. "Today" — Task database, filtered: Due = today OR Status = In Progress
3. "Overdue" — Task database, filtered: Due before today AND Status ≠ Done
[Continue for all sections]

Current frustrations or limitations:
[DESCRIBE ANYTHING THAT FEELS SLOW, CONFUSING, EMPTY, OR NOISY — or "none" if everything works]

How often I actually open this dashboard: [DAILY / A FEW TIMES A WEEK / RARELY]

Please evaluate the dashboard against these principles:
1. 5-Second Test: Can the most important task be identified within 5 seconds of opening?
2. Signal-to-noise: Are there sections I'm likely to skim past or ignore?
3. Action-orientation: Does each section lead to a decision or next action?
4. Cognitive load: Does the total number of sections add clarity or add overwhelm?

Please give me:
1. An assessment of each section: Keep as-is / Simplify / Move / Remove — with a 1-sentence reason
2. The two changes that would most improve usability this week (not next quarter)
3. One section I'd benefit from adding — based on what appears to be missing given my goal setup
4. A verdict: is this dashboard currently helping me act or helping me feel organised without acting?
```

### Prompt 6.3 — System Clutter Removal Prompt

**Purpose:** Identify components and sections you've built but don't need, and make a safe plan for removing them.

**When to use:** When your system feels heavy, complex, or like maintenance rather than a useful tool. Best used 4+ weeks after initial setup.

---

```
You are a minimalism consultant for personal productivity systems. My system has accumulated components I'm not sure I need. Help me identify what to cut safely.

Everything currently in my system (include ALL components, views, databases, rituals, and templates — even the ones not working):
[COMPREHENSIVE LIST — the more complete, the more useful]

What I use every day without thinking: [LIST — this is your core]
What I use only when I remember to: [LIST]
What I set up and haven't opened in 2+ weeks: [LIST]

My top goals right now: [1–3 GOALS]

Please:
1. Identify each component as: Essential / Useful but optional / Candidate for removal
2. For each "Candidate for removal": explain what I lose by keeping it (maintenance cost) vs. what I lose by removing it (if anything)
3. Suggest a safe removal order — starting with the lowest-risk cuts
4. Identify any component that looks abandoned but would be high-value if I simplified or restarted it
5. Create a short "minimum viable system" list — the 5–7 components I should keep even if I stripped everything else away
```

## Section 2: Workflow and Habit Optimization

---

### Prompt 6.4 — Kanban Workflow Design Prompt

**Purpose:** Get workflow stage suggestions tailored to your specific type of work — for a Kanban that actually matches how your work flows, not a corporate default.

**When to use:** Day 3, Session 2 — when setting up your Kanban. Also use when your current stages are causing confusion or tasks are sitting in wrong columns.

---

```
You are a personal workflow designer. I'm building a Kanban board for my Life OS and I want workflow stages that match how my work actually moves, not generic defaults.

My work type:
[DESCRIBE — e.g., "freelance content writer with 4 clients and a personal newsletter", "final-year student with coursework, a dissertation, a part-time job, and a personal fitness goal", "working professional with a side project and a content channel"]

What I'm managing across life areas:
[LIST THE CATEGORIES — e.g., "client work, own product dev, admin, personal goals, health routines"]

Current or default stages I'm working from:
[Inbox / Planned / This Week / In Progress / Waiting / Done]

What doesn't fit with the above:
[DESCRIBE — e.g., "I have a review phase", "feedback from clients is different from being blocked", "recurring admin is different from project tasks", "I have tasks waiting on self vs. tasks waiting on others"]

Please suggest:
1. A customised set of 5–7 stages for my Life OS Kanban, with a 1-sentence description of what goes in each
2. Any stage distinctions worth making (e.g., "Waiting: Self" vs. "Waiting: External")
3. Decision rules for the 3 trickiest transitions — when EXACTLY does a task move from one stage to another?
4. A WIP limit recommendation for my "In Progress" column — and a simple rule for enforcing it
```

### Prompt 6.5 — Habit Design Prompt

**Purpose:** Identify the best 2–3 process habits to track in your system — habits that create genuine leverage on your goals, not vanity metrics.

**When to use:** Day 3, Session 3 — when setting up a Streak or Habit Tracker. Also useful when you've been tracking habits but they don't seem to be helping.

---

```
You are a habits and ritual design coach. I want to identify the 2–3 daily habits that would have the highest leverage on my goals — and design them to be trackable and sustainable.

My top goals: [LIST 2–3 BRIEFLY]
My current system rituals: [WHAT I ALREADY DO — e.g., "morning planning 10 min", "weekly review", "daily task update in Notion"]
Rituals I keep skipping or doing inconsistently: [BE HONEST]

My daily available time for habits/rituals (not counting actual work): [X minutes — realistic, not aspirational]
My biggest obstacle to consistency: [e.g., "late starts", "days that start reactively", "high-travel periods", "low evening energy"]

For each habit you suggest, please provide:
1. Habit name and brief description
2. Why this habit has high leverage on my goals (not generic — explain the mechanism)
3. A simple yes/no completion criterion: "Done" means [SPECIFIC ACTION]
4. A minimum viable version for a difficult day — the version I can do in 5 minutes without failing the streak
5. The best time of day to anchor this habit, and what to anchor it to (habit stacking)

Rules for suggestions:
- Track PROCESS, not outcomes ("Did I do my morning planning?" not "Did I write 2000 words?")
- Maximum 3 habits — more than 3 habits tracked daily degrades all of them
- Do not suggest habits I already do consistently — only gaps
```

### Prompt 6.6 — Consistency Repair Prompt

**Purpose:** Diagnose why a specific system ritual has been breaking down and generate a targeted plan to rebuild it.

**When to use:** When a ritual you previously did well (weekly review, daily planning, logging) has been slipping for 2+ weeks and you're not sure how to get it back.

---

```
You are a ritual recovery specialist. I've fallen off a key system ritual and need to diagnose why and rebuild it in a way that actually sticks.

The ritual I've dropped: [NAME — e.g., "weekly review", "morning planning session", "Anti-To-Do logging"]
How consistently I was doing it before: [e.g., "every Friday for 6 weeks", "daily for 2 weeks"]
When it started slipping: [APPROXIMATE DATE OR EVENT — e.g., "after a busy travel week", "around the time I started a new project", "gradually over time with no clear trigger"]
How I've tried to restart it: [IF ANY ATTEMPTS — what happened?]

What the ritual was supposed to give me: [THE VALUE IT PROVIDED WHEN WORKING]
What happens when I skip it: [THE DOWNSTREAM EFFECT ON MY SYSTEM OR WEEK]

Please:
1. Identify the most likely reason the ritual broke down — be specific, not generic
2. Suggest whether to restart the ritual as-is, restart a simpler version, or redesign it
3. If redesigning: suggest one specific simplification that would make it more resilient
4. Suggest a specific re-entry point: exactly when and how to do it for the first time again (the exact first session, not a vague plan)
5. Suggest one environmental or calendar change that would make skipping harder — not willpower, but structure
```

### Prompt 6.10 — System Improvement Idea Generation Prompt

**Purpose:** Generate a list of targeted improvement ideas for a system that exists and is being used, but isn't delivering its full potential.

**When to use:** During retrospectives, or any time your system feels stale, heavy, or like it's not actually helping your output.

---

```
You are a productivity systems advisor. I want specific, implementable ideas for making my existing system more effective. I do NOT want to add complexity — I want to improve what's there.

My current system description (brief):
[DESCRIBE WHAT YOU HAVE — databases, views, rituals, key tools]

The symptoms I'm experiencing (even if I can't diagnose the cause):
[E.g., "I feel scattered in the morning", "I keep missing tasks that aren't in my Overdue view", "My dashboard has too much and I dread opening it", "I maintain the system but don't feel like it's helping my actual output"]

My constraints:
- I do not want to add new tools
- Any change should take less than 30 minutes to implement
- I want to modify what I have, not rebuild

Please generate 5 specific, actionable improvement ideas. For each:
1. Name the intervention clearly
2. Describe how to implement it in 2–3 concrete steps
3. Describe what problem or symptom it solves
4. Rate implementation effort: Quick win (< 30 min) / Medium (1–2 hours) / Significant redesign
5. Flag if this idea has any risk or downside

Do not suggest ideas that require rebuilding databases, adding new tools, or learning major new platforms. Focus on refinement.
```
