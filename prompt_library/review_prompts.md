# Review, Reflection & Reporting Prompts

## Building a Personal Productivity System with AI — Prompt Library

---

### When to Use This Category

Use these prompts to close the loop — at the end of a day, at the end of a week, after a difficult period, or when reviewing how a project is going.

Review prompts serve four purposes:
1. **Synthesise** — make sense of what actually happened vs. what you planned
2. **Diagnose** — find patterns and root causes (why things stall, what you avoid, what works)
3. **Communicate** — generate clear, credible status updates you can share
4. **Reset** — create a clean starting point for the next day, week, or cycle

### How to Use Any Prompt in This File

1. Pull your data first — from your Anti-To-Do Log, dashboard, or week's task status in Notion
2. Paste the data into the prompt template
3. Review the AI output critically — it reflects your input, so honest input = useful output
4. Translate insights into Notion: updated task statuses, next week's priorities, or a written status block

## Section 1: Daily Reflection

---

### Prompt 5.1 — End-of-Day Reflection Prompt

**Purpose:** Close the mental loop on the day — synthesise what happened, identify what carries forward, and set up a clean start for tomorrow.

**When to use:** At the end of every working day. Takes 5–10 minutes. More insightful than filling in a checklist because it prompts diagnosis, not just reporting.

---

```
You are my end-of-day thinking partner. Help me close out today honestly and set up tomorrow well.

Date: [DATE]
What I planned to do today: [LIST YOUR MORNING PLAN OR TOP 3 PRIORITIES]
What I actually completed: [LIST — can paste from Anti-To-Do or dashboard "Completed Today" view]
What didn't move: [TASKS THAT DIDN'T GET DONE — list them]
Focus quality today (1–10): [SCORE]
Energy at end of day (1–5): [SCORE]
One unexpected thing that happened: [IF ANYTHING — or "nothing unexpected"]

Please give me:
1. A 2-sentence honest summary of today — what was the real story of this day's work?
2. One insight about today that I should carry into tomorrow's planning
3. A suggested first task for tomorrow morning based on what didn't move today — something that's the highest-leverage "carry forward"
4. One pattern to watch: if today wasn't typical, what does it reveal about how I work?
5. One thing I should NOT do tomorrow that I attempted today — a pattern to recognise and avoid

Keep this practical and direct. No generic encouragement.
```

### Prompt 5.2 — Stuckness Diagnosis Prompt

**Purpose:** Help diagnose why a specific task or project has been avoided, delayed, or stalled — identifying the real reason rather than the surface one.

**When to use:** When the same task keeps rolling over from day to day, week to week. Also useful for projects that feel mysteriously hard to start. The most valuable prompt for breaking through chronic avoidance.

---

```
You are a behavioural productivity coach. I've been avoiding or delaying a specific task or project and I need honest help understanding why — and what to do about it.

The task or project I'm stuck on: [NAME AND BRIEF DESCRIPTION]
How long I've been avoiding it: [DAYS / WEEKS — be honest]
What I tell myself when I don't do it: [THE STORY YOU TELL — e.g., "I don't have enough time", "I'm waiting until I feel ready", "I need to do X first"]

What I know about this task:
- What "done" looks like: [DEFINITION OF DONE]
- Whether I have the skills to do it: [YES / MOSTLY / NOT SURE / NO]
- Whether I have everything I need to start: [YES / MOSTLY / MISSING: X]
- Whether anyone is waiting on this: [YES — WHO / NO]

Please analyse this honestly using these possible root causes:

1. Clarity problem: The task is too vague — I don't actually know what to do first
2. Skill gap: I'm worried I can't do it well enough
3. Emotional avoidance: This task is connected to something I fear (failure, feedback, vulnerability, conflict)
4. Overwhelm: The task feels too large and I don't have a starting point
5. Priority confusion: I'm not sure this task is actually important right now
6. External blocker: I'm genuinely waiting on something and haven't acknowledged it

Please:
1. Identify the most likely root cause(s) based on what I've described
2. Suggest one specific intervention for each root cause you identify
3. Define the one next physical action that would be hardest to say no to — something so small that avoiding it would require conscious effort
4. Ask me one question that might reveal a root cause I haven't considered
```

**Customisation notes:** This prompt is most powerful when you're brutally honest about "what you tell yourself." The AI can only work with what you give it.

## Section 2: Weekly Review and Reset

---

### Prompt 5.3 — Weekly Review Prompt

**Purpose:** Process and synthesise the week — identify what actually happened, the dominant patterns, and what one change would make next week better.

**When to use:** Day 4, Session 3 — as part of the weekly review. Repeat every Friday thereafter.

---

```
You are my weekly review partner. Help me run a structured review of this week and build a clear plan for next week.

Week of: [DATE RANGE]

Completed work (from my Anti-To-Do Log or Notion "Completed This Week" view):
[PASTE YOUR COMPLETED ENTRIES — one per line. Be exhaustive — include small wins.]

Tasks planned but not completed:
[LIST — for each, note why: Externally blocked / Actively avoided / Deprioritised consciously / Ran out of time]

Active project status:
[PROJECT 1]: [ON TRACK / AT RISK / STALLED] — [1 sentence why]
[PROJECT 2]: [ON TRACK / AT RISK / STALLED] — [1 sentence why]
[PROJECT 3]: [ON TRACK / AT RISK / STALLED] — [1 sentence why]

Attention allocation — actual vs. ideal:
[DOMAIN 1]: Ideal [%] → Actual [%]
[DOMAIN 2]: Ideal [%] → Actual [%]
[DOMAIN 3]: Ideal [%] → Actual [%]

One thing that was unexpectedly hard this week: [DESCRIBE]
One thing that went better than expected: [DESCRIBE]

Please:
1. Summarise the week in 3 sentences — what was the real output, and what was the dominant pattern?
2. Identify the single most significant time/attention misalignment this week
3. Identify the one incomplete task most likely to create problems if it stays incomplete into next week
4. Suggest one specific, actionable structural change for next week that addresses the biggest pattern
5. Ask me one question that would help me plan next week with more intention
```

## Section 3: Blocker and Pattern Diagnosis

---

### Prompt 5.5 — Project Blocker Diagnosis Prompt

**Purpose:** Identify why a specific project is stalled, at risk, or not progressing despite effort — and generate a concrete recovery plan.

**When to use:** When a project's health is "At Risk" or "Stalled" in your weekly review. Also use before a milestone check-in or stakeholder conversation.

---

```
You are a project health analyst. I have a project that is stalled or at risk and I need to diagnose it clearly.

Project: [PROJECT NAME]
Goal it serves: [PARENT GOAL]
Current status: [AT RISK / STALLED / BEHIND EXPECTATIONS]
Original deadline: [DATE]
Milestone I should be at by now: [NAME THE MILESTONE]
Milestone I'm actually at: [WHERE AM I HONESTLY]

What I've tried to do to move this forward: [LIST ACTIONS TAKEN]
What keeps happening instead: [DESCRIBE THE PATTERN — e.g., "I start, get interrupted, lose thread", "waiting on feedback that doesn't come", "I don't know what to do next"]
The last meaningful progress on this project: [DATE AND WHAT HAPPENED]

Please:
1. Identify the most likely root cause of the stall — be specific (not "you need to prioritise it better")
2. Suggest 2–3 targeted interventions that address the root cause
3. Recommend a revised milestone schedule: if I take action today, what's a realistic new timeline?
4. Identify if scope, timeline, or resources need to be renegotiated — and with whom
5. Suggest the ONE action I should take within the next 24 hours to break the current pattern
```

## Section 4: Progress Reporting

---

### Prompt 5.6 — Progress Update Generator Prompt

**Purpose:** Generate a clear, credible, professional progress summary you can share with a mentor, manager, accountability partner, or course instructor.

**When to use:** Any time you need to communicate your progress — for a performance update, a check-in with a mentor, or an accountability check. One of the highest-leverage prompts in the library for learners who work with others.

---

```
You are a professional communications specialist. I need to generate a clear, credible status update that I can share with [RECIPIENT: mentor / manager / accountability partner / course instructor].

Context:
- What I'm reporting on: [The project, goal, or period — e.g., "my Q1 content goal", "my system build", "Week 3 of the course"]
- Recipient: [MENTOR / MANAGER / ACCOUNTABILITY PARTNER / INSTRUCTOR]
- The level of detail appropriate: [BRIEF (3–5 sentences) / DETAILED (structured summary) / PRESENTATION-READY]

My progress this period:
- Completed: [LIST WHAT ACTUALLY HAPPENED]
- In progress: [LIST WHAT IS UNDERWAY]
- Blocked or delayed: [LIST WHAT DIDN'T HAPPEN AND WHY]
- Key decisions made: [ANY PIVOTS, SCOPE CHANGES, OR DIRECTION CHANGES]

Challenges I want to be honest about: [WHAT WENT LESS WELL THAN EXPECTED]
Achievements I want to make sure are visible: [WHAT I'M PROUD OF — be specific]

Next steps: [WHAT I'M DOING NEXT — with dates if relevant]

Please generate:
1. A short version (3–5 sentences) I can paste into a Slack message or email
2. A structured version with headings I could use in a written update or document
3. One improvement suggestion for how to frame this update to be most useful to my recipient
```

### Prompt 5.7 — Project Status Review Prompt

**Purpose:** Generate a clear status summary for a specific project — useful before a milestone review, meeting, or weekly update.

**When to use:** Any time you need to articulate where a project stands. Especially useful if the project has been vague or hard to describe clearly.

---

```
You are a project communications specialist. I need a clear status summary for one of my active projects.

Project: [PROJECT NAME]
Goal it serves: [PARENT GOAL]
Original deadline: [DATE]

Milestones completed: [LIST — with completion dates if known]
Milestones remaining: [LIST — with target dates]
Last meaningful progress: [DATE — and what happened]

Current blockers: [DESCRIBE — or "none currently"]
Upcoming tasks (next 7 days): [LIST 2–3]
Key risks: [ANYTHING THAT COULD DERAIL THE TIMELINE]

Please:
1. Rate this project's health: 🟢 On track / 🟡 At risk / 🔴 Stalled — with 1-sentence justification
2. Identify the single biggest risk to hitting the deadline
3. Suggest the one action that would most improve the project's outlook in the next 48 hours
4. Write a 3-sentence status summary I could share with a stakeholder or include in my weekly update
```

## Section 5: Monthly Retrospective

---

### Prompt 5.8 — Monthly Retrospective Prompt

**Purpose:** Run a deeper, system-level review of the past month — identifying patterns in how you work, which goals progressed, and what structural improvements to make.

**When to use:** Day 4, Session 3 (as a deeper review) — then once a month. More comprehensive than the weekly review.

---

```
You are a productivity system analyst running a 30-day retrospective. Help me understand the patterns of the past month and make meaningful structural improvements.

Month under review: [MONTH / DATE RANGE]

Goals progress:
[GOAL 1]: [% complete / milestone reached / stalled]
[GOAL 2]: [% complete / milestone reached / stalled]
[GOAL 3]: [% complete / milestone reached / stalled]

Work shipped this month (major outputs only):
[LIST THE 5–10 MOST SIGNIFICANT THINGS YOU COMPLETED]

System usage:
- Consistently used: [LIST COMPONENTS YOU ACTUALLY USED]
- Rarely used: [LIST COMPONENTS YOU OPENED BUT DIDN'T REALLY USE]
- Not used: [LIST COMPONENTS YOU SKIPPED ENTIRELY]

One pattern I noticed in how I work this month: [DESCRIBE — a recurring behaviour, good or bad]
One thing I'm proud of this month: [SPECIFIC]
One thing I most want to change about how I work next month: [SPECIFIC]

Please:
1. Identify 2–3 structural patterns from the month — not individual incidents but recurring themes (e.g., "morning planning happened but evenings were unstructured", "admin crowded out strategic work most weeks")
2. For each pattern: suggest one specific system change or ritual that would address it
3. Evaluate goal progress: are my active goals still the right ones? Flag any that should be archived or renegotiated.
4. Recommend 1–2 components to simplify or remove from my system based on the usage data
5. Set one clear focus or intention for next month — one sentence I can pin to my dashboard
```

### Prompt 5.9 — Anti-To-Do Summary Prompt

**Purpose:** Generate a polished weekly accomplishment summary from your Anti-To-Do entries — useful for performance reviews, accountability updates, or personal recognition.

**When to use:** As part of the weekly review. Paste entries from your Anti-To-Do Log in Notion.

---

```
You are a professional accomplishment writer. Please generate a clean accomplishment summary from my Anti-To-Do entries.

Anti-To-Do entries this week:
[PASTE ALL ENTRIES — in any format, just get them all in]

Please:
1. Write a 2–3 sentence summary capturing the main output and direction of this week's work
2. Organise by category (Work / Learning / Admin / Personal / Creative — use only categories relevant to my entries) and list 3–5 highlights per category
3. Identify any pattern in the type of work I completed this week (e.g., "mainly delivery work with no strategic or creative output")
4. Write one sentence I could use to describe this week's work to a colleague, mentor, or in a reflection log

Format: professional, clean, no filler language. Every sentence should contain a specific output or insight.
```
