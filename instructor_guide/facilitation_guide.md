# Facilitation Guide

## Building a Personal Productivity System with AI

---

### Overview

This guide provides practical facilitation instructions for running each session effectively. It covers room setup, energy management, common facilitation failures, and specific in-session moves.

---

### Before the Course: Facilitator Preparation

#### Technical Setup (Do Before Day 1)
- [ ] Create a demo Notion workspace using a fictional persona (recommend: "Alex, freelance UX designer, 31")
- [ ] Pre-populate Alex's workspace with a few fictional goals, projects, and tasks
- [ ] Test the Notion template link — duplicate it yourself and verify it works
- [ ] Create a ChatGPT account if you don't have one; test all 6 prompt library files
- [ ] Prepare a screen-sharing setup where learners can see both Notion and ChatGPT side-by-side
- [ ] Download all student task sheets and share them via the LMS or course folder

#### Room / Virtual Space Setup
- Breakout room capability for pair tasks (Zoom/Google Meet)
- Optional: a shared document or Miro board for group brainstorming
- A timer visible to learners (for timed tasks)
- A shared "questions parking lot" document for questions that arise mid-task

---

### Facilitation Rhythm: The 3-Part Session Pattern

Every session follows this pattern. Master it, and the course runs itself:

**1. Frame (5 min):** "Here's the problem you're about to solve. Here's why it matters. Here's how we'll solve it."  
**2. Show (10–15 min):** Demonstrate using Alex's workspace. Reach for the ChatGPT prompt. Edit the output visibly.  
**3. Build (30–40 min):** Learners build. You circulate (live) or monitor the chat/support channel (virtual). Give specific feedback based on what you see.

The frame is the most commonly skipped step. Don't skip it. If learners don't understand why they're building something, they build it passively and don't internalise it.

---

### Managing Pace

#### If a session runs long
- Cut the extension task, not the minimum viable task
- Cut the share-out if needed (move it to the next opening)
- Never cut the closing reflection — it's the moment that consolidates the day's learning

#### If a session runs short
- Use the extension task
- Run an unplanned pair share: "Turn to someone near you and show them what you just built"
- Use a discussion question from [`discussion_questions.md`](discussion_questions.md)

#### Speed distribution across learners
Most cohorts will have a 3x spread: some learners will finish tasks in half the time, others will need double. Strategies:
- Extension tasks for fast finishers (always available in the task sheets)
- Pair fast finishers with slow finishers for peer support (if the slow finisher is open to it)
- Instructor focuses support on learners who are blocked, not learners who are fast

### Day-by-Day Facilitation Notes

#### Day 1
- **Biggest risk:** Slow start because of tech setup. Build in 15 minutes of buffer.
- **Most important session:** Session 2 (Goal Design). If this goes well, the whole course is easier.
- **Watch for:** Learners who define tasks instead of goals ("Write 5 blog posts" is not a goal; "Grow my newsletter to 1,000 subscribers" is)
- **Facilitation move:** After the day 1 goal exercise, spot-check 3–4 screens. Read one goal out loud and ask the group: "Is this an outcome or an activity?"

#### Day 2
- **Biggest risk:** Attention budget exercise feeling confrontational. Frame it proactively: "This is data, not judgment."
- **Most important session:** Session 4 (Attention Budget). The gap between ideal and actual is the course's highest-impact insight moment.
- **Watch for:** Learners who haven't filled in their work session log — help them estimate from calendar history
- **Facilitation move:** After the attention budget comparison, ask: "What does this reveal that you already knew but hadn't said out loud?" Silence is fine here.

#### Day 3
- **Biggest risk:** Dashboard over-engineering. Learners spend 45 minutes perfecting a layout instead of building the essential views.
- **Most important session:** Session 2 (Dashboard Design). This is the session that makes the system feel real.
- **Watch for:** Learners who build linked databases that point to wrong databases — check their setup during the first 10 minutes
- **Facilitation move:** Do a "1-minute dashboard review" at the end: learners open their dashboard, you count to 5, they say whether they can find their top priority within 5 seconds.

#### Day 4
- **Biggest risk:** Capstone time pressure — presentations run long.
- **Most important session:** Session 4 (Capstone). Everything else in Day 4 prepares for this.
- **Watch for:** Learners who apologise for their system during the presentation. Redirect: "Don't apologise — tell me the reasoning behind each decision."
- **Facilitation move:** Keep a timer visible during capstones. At 8 minutes, give a gentle signal (raise a card or send a chat message: "2 minutes"). At 10 minutes, pause and move to Q&A.

### Facilitating AI Interactions

#### The most important facilitation move in this course

Every time you run a ChatGPT prompt in front of learners, follow this sequence:

1. **Read the prompt template aloud** — so learners understand what input they're providing
2. **Fill in the template visibly** — don't copy-paste; type it so learners see the customisation
3. **Submit and wait** — narrate what you expect: "I expect this to come back with..."
4. **Review critically** — read the output and find at least one thing to disagree with or change
5. **Edit visibly** — make the edit in the AI chat (follow-up prompt) or in the Notion entry
6. **Summarise** — "Here's what I kept, here's what I changed, here's why"

This sequence prevents learners from treating AI output as authoritative. It models the expert workflow: use AI as a collaborator, not an oracle.

### Handling Difficult Situations

#### Learner who is visibly overwhelmed
- Private message: "Where are you stuck? Let's simplify."
- Offer: "Let's skip the extension task and make sure the minimum version is solid."
- If they're blocked on knowing their goals: give them a starter goal ("Improve my professional reputation in my current field") and tell them they can change it tomorrow.

#### Technical failure (Notion down, ChatGPT unavailable)
- Use the task sheets — most tasks can be completed on paper or in a Google Doc if tools fail
- If Notion is unavailable, complete the planning tasks in Google Docs/Sheets
- If ChatGPT is unavailable, use the prompt library as a guided question set and have learners answer the questions themselves (this is often more powerful anyway)

#### Learner who questions the value of the system
- "Why should I do this when I can just use a to-do app?"
- Acknowledge: "You can. But to-do apps don't show you goals, don't link projects, don't track attention allocation. It's the difference between a list of chores and a strategy."
- If they persist: "Try it for 30 days. The question isn't whether the system works — it's whether it works for you."

---

### Between-Day Instructor Checklist

After each day, do a quick review:

- [ ] Which learners are behind? Plan to check in first thing next morning.
- [ ] Which concepts generated the most confusion? Adjust the next day's framing.
- [ ] Any prompts that produced unexpectedly poor AI outputs? Note and prepare a revised version.
- [ ] Any technical issues to resolve before tomorrow?
- [ ] Anything about pacing that needs adjustment?
