# Learning Outcomes

## Building a Personal Productivity System with AI

---

### Terminal Outcome (Course-Level)

By the end of this course, each learner will have **designed, built, and tested a personal AI-assisted productivity operating system** that they can begin using immediately after the course ends.

---

### Day-by-Day Learning Outcomes

---

#### Day 1: Foundation

**By the end of Day 1, learners will be able to:**

1. Define their top 3–5 life and work goals with specific, observable success criteria — using a structured AI prompt
2. Translate each goal into one or more active projects with clear scopes
3. Break a project into actionable tasks using a structured AI prompt
4. Distinguish between outcomes (goals), deliverables (projects), and actions (tasks)
5. Set up a working Notion workspace with a Goals Database, Projects Database, and Task Database — all linked together
6. Populate their workspace with at least one real goal, two active projects, and ten or more tasks

**Transformation Statement:**  
*On Day 1, learners go from scattered intentions to a structured, visible system of goals and tasks.*

---

#### Day 2: Prioritization

**By the end of Day 2, learners will be able to:**

1. Define milestones for each project and sequence them in reverse-planning order from the end goal back to today
2. Assign priority tiers (High / Medium / Low) to tasks using the Eisenhower-flavored AI prioritization prompt
3. Maintain an Anti-To-Do List that captures completed work and counters to-do guilt
4. Track work sessions using the Work Session Log template — time, project, output
5. Create an Attention Budget that maps desired focus percentages across domains (e.g., 40% project A, 30% study, 20% admin)
6. Identify gaps between their intended vs. actual time allocation using AI-generated analysis

**Transformation Statement:**  
*On Day 2, learners move from reacting to tasks to consciously choosing where their time and attention go.*

---

#### Day 3: Dashboard & UX

**By the end of Day 3, learners will be able to:**

1. Build a customised Kanban board with workflow stages tailored to their work style
2. Create a Master Dashboard inside Notion that aggregates filtered views of tasks, projects, and streaks
3. Set up a Streak Tracker to monitor daily consistency on their top 2–3 habits or routines
4. Apply usability principles to their dashboard (reduce noise, increase signal)
5. Design at least two custom filtered views — e.g., "Today's Priorities" and "This Week's Deliverables"
6. Evaluate their system's UX using a self-audit checklist driven by AI feedback

**Transformation Statement:**  
*On Day 3, learners turn a functional database into a control panel they actually want to open each morning.*

---

#### Day 4: AI Review & Capstone

**By the end of Day 4, learners will be able to:**

1. Run a complete daily AI planning session using a structured prompt sequence (context → priorities → blockers → plan)
2. Use an AI overload triage prompt to make fast decisions when their list is too long
3. Complete an end-of-day reflection using the daily check-in template
4. Run a full weekly review ritual — what shipped, what didn't, why, what to adjust
5. Conduct a system retrospective — identify what's working, what's not, and propose one concrete improvement
6. Present their complete productivity system to an audience or evaluator and explain the reasoning behind each design choice

**Transformation Statement:**  
*On Day 4, learners close the loop — they now have a system AND a set of rituals to maintain and improve it indefinitely.*

### Skills Taxonomy

| Skill Category | Skills Developed |
|----------------|-----------------|
| **AI Prompting** | Goal design prompts, task breakdown, prioritization, planning, reflection, system improvement |
| **System Design** | Database architecture, linked views, property design, filtering logic |
| **Planning & Prioritization** | Reverse planning, milestone sequencing, priority tiering, anti-to-do |
| **Time Awareness** | Attention budgeting, work session tracking, actual vs. ideal effort analysis |
| **Dashboard Design** | Kanban setup, view filtering, streak tracking, visual signal design |
| **Reflection Practice** | Daily check-in, weekly review, retrospective methodology |
| **Communication** | Capstone presentation, system walkthrough, design rationale |

---

### What This Course Does NOT Cover

To manage scope clearly:

- This course does not cover Notion integrations, APIs, or automations
- This course does not require coding of any kind
- This course does not cover project management for teams (only individuals and small team leads)
- This course does not use paid AI tools — all prompts work with ChatGPT free tier
- This course does not produce generic templates — everything is built around the learner's actual goals
